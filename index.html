Key Terms for Programming with JavaScript

JavaScript is an object oriented scripting language and as such employs the standard programming features and concepts listed below. All of the following will be discussed in further detail in the following sections but to introduce them quickly:

Statements

The coding equivalent of a sentence – one complete section of code that performs an action.

Like a sentence we need to use punctuation to indicate where one ends and the next begins - statements use a semi-colon for this instead of a full stop.

Statements can get quite long and complicated, but a very simple one might look like this:

x = 16;

This statement is saying the variable named x now is equal to 16.

Variables

Variables are containers used to store and retrieve values.

The following two statements define and then set the value of a variable:

let x;
x = 16;

The first statement is saying create a new variable named x and the second statement is setting x's value to 16.

Functions

Functions execute a list of statements, and often they are used to group these actions together in a specific order.

An empty function might look like this:

function exampleFunction() {

}

Parameters

One or more comma separated variables that are written in between a function's parentheses. Parameters change how a function works each time it runs. Some functions use parameters, but not all do.

A function with two parameters might look like this:

function exampleFunction(myParameter1, myParameter2) {

}

Block

A section of code that is run together – defined by using curly brackets.

This example function's block includes three separate statements that will then execute in the order written every time the function is run:

function exampleFunction(myParameter1, myParameter2) {
  let x = myParameter1 + myParameter2;
  x *= 2;
  return x;
}

Objects

JavaScript objects are collections of named information which we can use methods to interact with in a range of ways.

Most types of variables and all HTML elements are objects which allows us to dynamically change them.

In the example below the text value "patrick" becomes an object when it is assigned to myName variable. Once it becomes an object can we then run the toUpperCase() method which will return the text in all caps:

let myName = "patrick";
myName.toUpperCase();

Methods

Methods are built in functions that objects can run – they are used by typing the object name followed by a . and then the desired method and a set of brackets.

These two statements declare myName as a variable and then run the toUpperCase() method on that variable object:

let myName = "patrick";
myName.toUpperCase();

Naming in JavaScript w/ camelCase

When naming variables and functions it is standard practice to use camelCase - this refers to a style of writing phrases with multiple words without using spaces. It involves starting the phrase with a lower case letter and using an uppercase letter at the start of each subsequent word - eg: thisIsWrittenInCamelCase.

You can choose whatever name you want for variables and functions as long as it doesn't start with a number or match a reserved keyword. Best practice is to be descriptive with your names, for example if I was to write a function that multiplies two numbers together calling it something like multiplyTwo is much better than something generic like myFunction.

JavaScript and the HTML DOM

When a browser turns your code into a web page it creates a dynamic model called the HTML DOM (Document Object Model). The DOM takes the form of a tree of objects, as you can see in the image here.

As HTML elements in the DOM are objects JavaScript can be used to manipulate them, allowing you to dynamically change web pages.

JavaScript can dynamically change all of the HTML attributes on the page.

JavaScript Statements

JavaScript statements are single instructions to do something. Depending on what you're trying to do they might include:

  • variables
  • values
  • operators
  • keywords
  • and/or comments

While the actual parts may vary they always end with a semi-colon.

It is common for JavaScript statements to look something like this:

// this changes the text inside the element with the id of testID
document.getElementById("testID").innerHTML = "Hello World";

or this:

// this returns the value of the variable x
return x;

or this:

// this calls the function called myFunction with arguments of 3 and 12
myFunction(3, 12);

Like a sentence in any other language there are many forms the structure or syntax of a statement might take. The important thing is that they express a single instruction even if that instruction might have several stages. While not essential to its operation it is standard practice to end your statement using a semi-colon and to start a new line in your code before starting the next one so you can easily read where one ends and the new statement begins.

JavaScript Functions

JavaScript functions are blocks of code, commonly made up of multiple statements, that 'do' something - once defined functions can be called as often as you like.

Below is an example of a function definition:

function myExternalFunction() {
  document.getElementById("demo3").innerHTML = 'Zzzzz';
}
  • First we use a keyword to tell the computer that we're defining a function - the keyword is the word function
  • Next we give our function a name - in this case myExternalFunction. You can call your function whatever you want be each name should be unique in your project.
  • Next we write a pair of parentheses like this () - if we want to declare some parameters (see below) this is where we'll put them
  • Finally we write some curly brackets to contain the code block that we want to run when the function is executed

Function parameters set what arguments the function can receive in the form of values, which are then stored as local variables. Like the unique name you give your function, parameter's names are chosen by you and can be whatever you like as long as no two parameters share the same name. Functions often don't have any parameters, such as in the example above, but if they do multiple parameters can be added at the same time by separating them with commas.

This might seem like a lot of moving parts but it's not so bad in practice - for example let's read the function written below:

function myCalculator(a, b, c) {
  let sum = a + b + c;
  document.getElementById('demo5').innerHTML = sum;
}
  • First I defined it as a function that uses the name myCalculator and it takes in three separate parameters called a, b and c
  • When this function is excuted it will run the code inside the curly brackets {} - in this case there's two statments listed inside the block
  • The first statement creates a variable called sum (more on variables in a section below) and sets its value to the result of adding the three parameters together using the expression a + b + c.
  • The second statment looks at our document (the DOM), within that it then finds the element which has the id of demo5, then it changes that element's innerHTML (more on properties in a section below - tl;dr its what is written inside the HTML element) to equal whatever sum was set to in the previous statement.

One of the reasons you might want to do this is to cut down on writing out code - you can see an example using the same addThree function on both buttons below. I only need to define my function once and then I can call that function as many times as I want by using its name followed by the operator (). In this case I can also get different results by changing out the parameters - the first calls the function as addThree(3, 2, 1) and the second calls it as addThree(5, 4, 3).

JavaScript Variables

A JavaScript variable is an object we can create with its own specific name that can be used as a container to store a value, like some text or a number. Variables need to be defined the first time you reference them using a keyword, in this case let, followed by a unique name of your choice, such as in the example bellow:

let myHairColour;

When defined like this myHairColour doesn't contain any value and when queried will return:

undefined

While you can assign a value in later statements often you'll want to initialise a variable with a value which can be done using the equals operator as seen below:

let myHairColour = "brown";

Like functions you can only define variables once, but once you have defined them using their unique name you can refer to them or reasssign their value as much as you want, such as the example below. I'm initialising my variable with the value "brown" then changing it to "blonde" in the next statement (note I only use let once when I define the variable, I don't need to use it when I'm changing that variable's value).

let myHairColour = "brown";
console.log(myHairColour); // this query would return the value "brown"
myHairColour = "blonde";
console.log(myHairColour); // this query would return the value "blonde"

JavaScript is run sequentially, statement by statment top to bottom - if we check the value of the variable at different stages it will give us different results.

Variables can store many different kinds of data and allow for different functionality (see methods section below) based on what type of data they are storing. For now let's start with the two most basic types: numbers and strings.

Numbers - Variables can store two types of numbers, whole numbers like 23 (referred to as integers or ints) or numbers with a decimal place like 3.142 (referred to as floating point numbers or floats). JavaScript will work out which is which when you assign the number to a variable and there are some times where this distinction is important, such as when using an array as we did for our filter in week 5. A helpful reminder is thinking of the difference between an elevator (integer) and a slide (float). An elevator can only travel a floor at a time (a whole number) while slide can travel in a continuous motion (a decimal number).

// this is an int variable
let myAge = 34;
// this is a float variable
let coffesDrunk = 1.4;

Strings - Variables can also store text, one section of text is called a string and can be as long or short as you'd like. Unlike numbers strings always need to be wrapped inside either single or double quote marks.

// this is NOT an acceptable string - the code will instead look for something like another variable or function called brown
let myHairColour = brown;
// this is an acceptable string
let myHairColour = "brown";
// so is this
let myEyeColour = 'blue';

It's good practice to stick to using one or the other type of quote marks for defining strings as sometimes you'll need to nest them. In the example below I wanted to use a string inside of some inline JavaScript but because it's part of an attribute my browser will read the end of the value at the second " it sees. By alternating the type of quotes used the HTML will be able to understand the difference.

// this code can read the difference between the string and the attribute
<button onclick="printString('mystring')">print</button>
// this code thinks the onclick attribute ends after the (
<button onclick="printString("mystring")">print</button>

JavaScript Objects - Properties & Methods

In JavaScript an object is a specific data type that refers to a collection of information accessible under a particular name. Variables and HTML elements are the most common type of object you'll deal with as you start to learn JavaScript. The information stored in an object is accessible as a property and they also inherit a number of generic functions called methods. We'll keep it simple to start with when working with objects but we'll dip into some more advanced uses as we progress this semester.

Object Properties

As mentioned this is how we access the information stored in an object - in the example below we're able to find the innerHTML property of a HTML element.

// this line uses a method (explained below) to find a HTML element in our HTML DOM and then saves that element as a variable
let testObject = document.getElementById("testID");
// this line then finds the innerHTML property of the testObject variable and saves it to its own variable
let testObjectString = testObject.innerHTML

As shown here properties are accessed by writing the name of the object follow by a . then the name of the property. In this case the innerHTML property will return whatever is written inside of the element, that might be a string of text or even other HTML elements. All HTML attributes are accessible as properties using the same name as the attribute and there are other useful properties built in to all HTML element objects.

Object Methods

JavaScript methods are predefined functions that are stored as a property of a JavaScript object - like functions you might write yourself methods are called by using their name (written in camel case) followed by a set of brackets that may or may not need argument values placed inside. Unlike functions they are can only be called in relation to an object - as shown in the example below this relationship is defined by naming the object first followed by the method with a dot in between:

document.getElementById("testID");

In this case we are calling the getElementById() method on the document object. document is a predefined object that represents the HTML DOM - running the getElementById() method on it with an argument value of "testID" will look through each HTML element currently part of the HTML DOM and return the first element with an id attribute value of "testID". By itself this method won't do anything once it finds that element however if we were to add the following:

document.getElementById("testID").innerHTML = "Hello World";

we could change the innerHTML property (what's written inside that element) to the text "Hello World" by using the equals operator. .

To reiterate the above statement is first calling the getElementById() method on the document object and then changing the text inside the resulting element to "Hello World" by accessing the innerHTML property.

Different objects contain different methods and properties but while starting out using the getElementById() method to find an element and then changing that element's properties is one of the most common things you'll want to do.
